# 360doegaku AI開発共通テンプレート

## 0. 作業開始前の確認

作業を開始する前に、以下のドキュメントを確認すること。

- PROJECT.md
- ARCHITECTURE.md
- ROADMAP.md

今回の作業内容が、プロジェクト全体の設計方針と矛盾していないことを確認してから実装を開始する。

---

## 1. 作業名

ここに今回の作業名を記入する。

例：

フェーズ2-4：自動回転の停止

---

## 2. 目的

今回の作業で達成したい内容を記入する。

例：

球体の自動回転を停止し、360度空間の基礎表示を静止状態にする。

---

## 3. 現在の状態

作業前の状態を記入する。

例：

- 球体の内側が表示されている
- ワイヤーフレームが右から左へ自動回転している
- Three.js関連コードは役割ごとに分割済み

---

## 4. 修正対象

変更してよいファイルを明記する。

例：

- src/renderer/src/renderer.ts

指定されていないファイルは変更しないこと。

---

## 4-1. 修正範囲

指定したファイルのみ変更すること。

他のファイルにも修正が必要と思われる場合は、勝手に変更せず、その理由を報告すること。

---

## 5. 実装内容

今回実装する内容を具体的に記入する。

例：

- アニメーションループ内の球体回転処理を削除する
- レンダリング処理は維持する
- 球体の内側表示は維持する

---

## 6. 必須条件

必ず守る条件を記入する。

例：

- TypeScriptエラーを発生させない
- 現在の見た目を回転停止以外は変更しない
- import/exportを壊さない
- npm run typecheckを通す
- npm run lintを通す

---

## 7. 変更禁止

今回変更してはいけない内容を記入する。

例：

- 新機能の追加
- OrbitControlsの追加
- マウス操作の追加
- ファイル構成の変更
- 変数名の変更
- 大規模なリファクタリング
- package.jsonの変更
- CSSの変更
- ライブラリの追加
- コメントの大量追加

---

## 8. 作業後の確認

作業後に確認する項目を記入する。

例：

1. npm run typecheck
2. npm run lint
3. npm run devのビルド成功
4. 球体の内側表示が維持されている
5. 球体が自動回転しない
6. 変更対象以外のファイルが変更されていない

---

## 9. 作業結果の報告

作業後は、以下を簡潔に報告すること。

- 変更したファイル
- 変更内容
- 実行した確認コマンド
- 確認結果
- GUI環境の制約など、確認できなかった項目

---

## 10. 共通ルール

- 一度の作業では一つの目的だけを扱う
- 指示されていない機能を追加しない
- 指示されていないファイルを変更しない
- 現在の動作を必要以上に変更しない
- 不明点がある場合は推測で実装せず、作業前に確認する
- 既存コードの命名と構造を尊重する
- 最小限の変更で目的を達成する
- 作業後に差分を確認する
- エラーがある場合は隠さず報告する

---

## 11. 出力形式

作業終了後は、以下の形式で報告すること。

### 変更したファイル

- （ファイル名）

### 実装内容

- （実装した内容）

### 実行した確認コマンド

- （実行したコマンド）

### 確認結果

- （確認結果）

### 注意事項

- （GUI環境などにより確認できなかった内容や補足事項）
